## 🏆 Scoring Pillars: Parameter Impact Breakdown

Your final score is the sum of all 6 parameters. No category is ignored. Optimize strategically based on their impact weightage:

### 🔴 High Impact (Most Crucial)
*   **Code Quality:** Clean, readable, and well-structured code. Keep files modular, avoid spaghetti logic, and leverage custom CSS variable setups.
*   **Problem Statement Alignment:** Directly targets the core challenge, student burnout stress causes (mock tests, backlogs, parents), and offers concrete coping solutions.

### 🟡 Medium Impact (Differentiating Factors)
*   **Security:** Safe practices, avoiding vulnerabilities. Secure your API keys on the server side (Next.js Route Handlers) and filter input schemas.
*   **Efficiency:** Optimal use of CPU execution time and memory limits. Limit bundle sizes, and use fast vector inline SVGs for rendering metrics instead of heavy rendering engines.

### 🟢 Low Impact (Tie-Breaker Points)
*   **Testing:** Easily testable & maintainable code. Code should support isolated props and fallback test parameters.
*   **Accessibility:** Usable for diverse users and environments (e.g. Web Speech dictation for hands-free operations, high-contrast theme variations).

### Step 3: Component Assembly
